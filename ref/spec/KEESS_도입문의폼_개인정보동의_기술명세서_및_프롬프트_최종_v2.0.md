### 1-2. 법적 근거

1. **개인정보 보호법 제15조 제1항** — 담당자명+연락처/이메일 수집에 적법 근거 필요. 현재 4개 폼은 동의도 고지도 없음.
2. 제15조①4호(계약 체결 전 조치) 경로는 처리방침 공개(제30조)가 전제이나, KEESS 처리방침은 미제공(placeholder) 상태 → 성립 불가.
3. 홈 폼은 동의를 받고 서브 폼은 받지 않는 불일치 구조 → "인지하고도 누락"으로 해석될 최대 리스크 형태.
4. 다운로드 모달의 매월 이메일 발송은 지속적 발송으로 광고성 정보 소지(정보통신망법 제50조 — 사전 동의 필수, 매 발송 시 수신거부 수단 표기). **문제는 발송 자체가 아니라 선택권 없이 전원 수신처럼 고지된 형태**이며, 옵트인(선택 동의)으로 전환하면 적법하게 유지 가능.
5. **동의 강요 금지(제22조)** — 매월 수신을 필수 동의에 포함 금지. 선택 동의로 분리하고 미동의 시에도 다운로드는 가능해야 함.

### 1-3. 전략 확정

| 판단 | 내용 | 근거 |
| --- | --- | --- |
| 필수 동의 4곳 즉시 적용 | 수집 자체의 적법성 확보. **오픈 후 소급 수집 불가**이므로 오픈 전 필수 | 제15조·제22조 |
| 다운로드 모달 '매월 리스트 수신'은 옵트인으로 유지 | 배너를 선택 동의 연동 문구로 교체하고 선택 동의 1개 추가. 원하는 사용자만 수신 | 정보통신망법 제50조 사전 동의 충족 |
| 발송 미운영 확정 시 히든 스위치 | 배너+선택 동의를 **한 곳의 설정(`optional: null`)으로 동시 비노출**. 배너만 남고 동의만 사라지는 위법 상태가 구조적으로 불가능하도록 묶음 처리 | 운영 확정 전 유연성 확보 |

## 2. 공통 동의 컴포넌트 명세

### 2-1. 기준 마크업 (홈 v18 확정본 재사용)

홈 상담 폼의 동의 UI 구조·클래스를 그대로 재사용한다. 신규 스타일 작성 금지.

```
.consent-group
 └ .consent                       ← 체크박스 행
    └ label.consent-main > input[type=checkbox] + span
       └ b.c-tag.req-tag "필수"   (선택 동의는 b.c-tag.opt-tag "선택")
    └ button.consent-view "전문 보기" (data-c="{키}")
 └ .consent-text#{키}-text        ← 전문 아코디언
    └ .ct-inner  (목적 / 항목 / 보유기간 / 거부권리 순)
```

### 2-2. 동작 규칙

1. "전문 보기" 클릭 시 해당 `.consent-text` 아코디언 토글 (`aria-expanded` 반영)
2. 필수 동의 미체크 제출 시: 제출 차단 + 해당 동의 행 직하단에 기존 `.err` 패턴으로 "필수 동의 항목을 확인해 주세요." 표기
3. **선택 동의는 미체크여도 제출·다운로드 가능** (검증 대상 아님)
4. 체크박스 id·전문 키는 폼별 접두어로 전역 유일 (`ld-`/`hrd-`/`dl-`/`ct-`) — `/content`는 모달 2개 공존하므로 필수
5. 동의 문구는 `CONSENT_TEXTS` 단일 상수 객체에서만 관리. 폼별 엔트리는 `{ required: {...}, optional: {...}|null }` 구조.
   - `optional`이 **null이면 선택 동의 행과 연동 배너를 함께 렌더하지 않는다** (히든 스위치)
   - 1차 설정: download만 `optional` 채움, 나머지 3개 폼은 null

## 3. 적용 지점별 명세 (4곳)

### 3-1. `/leadership` 도입 문의 폼

- 삽입 위치: "문의 내용" textarea 아래, [도입 문의 보내기] 버튼 위
- 구성: 필수 동의 1개 (id `ld-agree1`, 키 `ldpriv1`), `optional: null`
- 전문 가안:

> **개인정보 수집·이용 안내 (필수)**
> KG에듀원 KEESS는 「개인정보 보호법」 제15조에 의거하여 도입 문의 처리를 위한 개인정보 수집·이용 동의를 받습니다.
> **1. 수집·이용 목적** — 리더십·조직 솔루션 도입 문의의 접수·상담 및 회신
> **2. 수집 항목** — (필수) 담당자명, 연락처, 회사명
> **3. 보유 및 이용 기간** — 동의일로부터 1년간(보유목적 달성) 또는 삭제 요청 시 지체 없이 파기
> **4. 동의 거부 권리** — 동의를 거부할 권리가 있습니다. 다만 거부 시 도입 문의 서비스 이용이 제한될 수 있습니다.

### 3-2. `/hrd` 도입 문의 폼

- 3-1과 동일 구조 (id `hrd-agree1`, 키 `hrdpriv1`), `optional: null`
- 전문 가안: 목적만 "HRD 통합 솔루션 도입 문의의 접수·상담 및 회신"으로 교체

### 3-3. `/content` 전체 과정 리스트 다운로드 모달 — 필수 + 선택(옵트인) 2단

- **배너 문구 교체** (선택 동의와 연동):
  - 기존: "입력하신 메일로 매월 갱신되는 최신 리스트도 함께 안내드립니다."
  - 변경: "아래 수신 동의(선택)에 체크하시면 매월 갱신되는 최신 리스트를 이메일로 보내드립니다."
  - 모달 헤더의 "Excel · 매월 갱신 · 8,000여 과정" 서브텍스트는 리스트 자체의 갱신 주기 설명이므로 유지
- 삽입 위치: "이메일" 필드 아래, [리스트 다운로드] 버튼 위
- 구성:

| 구분 | 항목 | 검증 | id / 키 |
| --- | --- | --- | --- |
| 필수 | 개인정보 수집·이용 안내 (리스트 제공 목적) | 미체크 시 제출 차단 | `dl-agree1` / `dlpriv1` |
| 선택 | 매월 최신 과정 리스트 이메일 수신 동의 | 미체크여도 다운로드 가능 | `dl-agree2` / `dlpriv2` |

- **필수 전문 가안**: 목적 "전체 과정 리스트 파일 제공 및 관련 안내" / 항목 "(필수) 담당자명, 회사/기관, 이메일" / 기간·거부권리 3-1 준용 (거부 시 "리스트 다운로드 이용이 제한될 수 있습니다")
- **선택 전문 가안**:

> **매월 최신 과정 리스트 이메일 수신 동의 (선택)**
> **수집·이용 목적** — 매월 갱신되는 KEESS 전체 과정 리스트의 이메일 발송
> **수집 항목** — 이메일
> **보유 및 이용 기간** — 수신 동의 철회 시까지
> 동의하지 않아도 리스트 다운로드는 이용하실 수 있습니다. 발송되는 이메일에서 언제든 수신을 거부할 수 있습니다.

- **히든 스위치**: 배포 전 매월 발송 업무 미운영으로 확정되면 `CONSENT_TEXTS.download.optional = null` 한 줄로 전환 → 배너와 선택 동의 행이 **함께** 비노출되고 필수 동의만 남는다. 배너와 선택 동의는 반드시 같은 설정에 묶여 있어야 하며 개별 토글을 만들지 않는다(배너만 노출되는 위법 상태 방지).
- **발송 운영 조건(개발 외 운영 요건)**: 선택 동의자에게만 발송, 매 발송 메일 하단에 무료 수신거부 수단 표기 (정보통신망법 제50조)

### 3-4. `/content` 콘텐츠 도입 문의 모달

- 삽입 위치: "필요한 콘텐츠·과제" textarea 아래, [문의 보내기] 버튼 위
- 구성: 필수 동의 1개 (id `ct-agree1`, 키 `ctpriv1`), `optional: null`
- 전문 가안: 목적 "콘텐츠 솔루션 도입 문의의 접수·상담 및 회신", 항목 "(필수) 담당자명, 회사/기관, 이메일", 기간·거부권리 3-1 준용

### 3-5. 제출 데이터 규칙 (4곳 공통)

- 제출 payload: `privacy_agreed: true`, `agreed_at: ISO8601` — 동의 사실 입증 기록(입증 책임은 처리자에게 있음)
- 다운로드 모달만 `marketing_agreed: true|false` 추가 전송 (선택 동의 렌더 시). `optional: null` 전환 시 미전송
- 입력값(이름·연락처·이메일 등)은 GA4 등 분석 도구로 전송 금지 (측정설계서 9.4절 원칙 유지)

## 6. 클로드 코드용 빌드 프롬프트

```
KEESS 웹사이트의 개인정보 수집 폼 4곳에 동의 컴포넌트를 추가하는 작업이다.
법적 배경: 이 4개 폼은 개인정보(담당자명, 연락처/이메일)를 동의 절차 없이 수집하고
있어 개인정보 보호법 제15조 위반 소지가 있다. 홈 교육상담 폼에 이미 구현된 동의
컴포넌트를 기준으로 재사용한다. 이 4곳 외 다른 페이지·폼의 문구와 구성은 절대
변경하지 않는다.

[작업 원칙]
1. 홈 상담 폼의 동의 컴포넌트(.consent-group / .consent / .consent-main /
   .consent-view / .consent-text / .ct-inner 구조, 전문 보기 토글 JS, 필수 미체크
   시 제출 차단 검증)를 공통 컴포넌트로 추출해 재사용한다. 신규 스타일 금지.
2. 동의 전문 텍스트는 CONSENT_TEXTS 상수 객체 하나로 분리한다. 폼별 엔트리는
   { required: {...}, optional: {...}|null } 구조로 하고, optional이 null이면
   선택 동의 행(과 그에 연동된 배너)을 렌더하지 않는다. 문구는 추후 정보보호팀
   승인으로 교체될 가안이므로 이 객체만 수정하면 전 폼에 반영되어야 한다.
3. 홈 상담 폼과 부정훈련 신고 모달의 기존 동의 구조는 절대 수정하지 않는다.
4. 체크박스 id는 페이지 접두어로 유일성 보장: ld- / hrd- / dl- / ct-
   (/content는 모달 2개가 공존하므로 충돌 금지)

[적용 지점 4곳]
A. /leadership 최하단 도입문의 폼: "문의 내용" 아래·제출 버튼 위에 필수 동의 1개
   (id: ld-agree1, 전문 키: ldpriv1). optional: null.
   필수 미체크 제출 시 차단 + "필수 동의 항목을 확인해 주세요." 에러를 동의 행
   직하단에 표기.
B. /hrd 최하단 도입문의 폼: A와 동일 (id: hrd-agree1, 키: hrdpriv1). optional: null.
C. /content "전체 과정 리스트 다운로드" 모달:
   ① 상단 안내 배너 문구를 다음으로 교체한다:
      "아래 수신 동의(선택)에 체크하시면 매월 갱신되는 최신 리스트를 이메일로
      보내드립니다."
      (헤더의 "Excel · 매월 갱신 · 8,000여 과정" 서브텍스트는 유지)
   ② 이메일 필드 아래·[리스트 다운로드] 버튼 위에 순서대로 추가:
      - 필수 동의 (id: dl-agree1, 키: dlpriv1) — 미체크 시 제출 차단
      - 선택 동의 (id: dl-agree2, 키: dlpriv2, opt-tag "선택") — 미체크여도
        다운로드 가능
   ③ 이 배너는 CONSENT_TEXTS.download.optional 존재 여부에 종속시켜, optional을
      null로 바꾸면 배너와 선택 동의 행이 함께 사라지고 필수 동의만 남아야 한다.
      (매월 발송 업무가 미운영으로 확정될 경우 이 한 줄로 히든 처리하기 위함.
      배너만 노출되고 동의만 사라지는 조합은 구조적으로 불가능해야 한다)
D. /content "콘텐츠 도입 문의" 모달: "필요한 콘텐츠·과제" 아래·버튼 위에 필수
   동의 1개 (id: ct-agree1, 키: ctpriv1). optional: null.

[동의 전문 가안 — CONSENT_TEXTS에 그대로 수록]
공통 구조: 도입부(「개인정보 보호법」 제15조 근거 고지) + 목적 / 수집 항목 /
보유·이용 기간 / 동의 거부 권리 4단.
- ldpriv1: 목적 "리더십·조직 솔루션 도입 문의의 접수·상담 및 회신",
  항목 "(필수) 담당자명, 연락처, 회사명",
  기간 "동의일로부터 1년간(보유목적 달성) 또는 삭제 요청 시 지체 없이 파기",
  거부권리 "동의를 거부할 권리가 있습니다. 다만 거부 시 도입 문의 서비스 이용이
  제한될 수 있습니다."
- hrdpriv1: ldpriv1과 동일, 목적만 "HRD 통합 솔루션 도입 문의의 접수·상담 및 회신".
- dlpriv1(필수): 목적 "전체 과정 리스트 파일 제공 및 관련 안내",
  항목 "(필수) 담당자명, 회사/기관, 이메일", 기간 동일,
  거부권리 말미 "거부 시 리스트 다운로드 이용이 제한될 수 있습니다."
- dlpriv2(선택): 제목 "매월 최신 과정 리스트 이메일 수신 동의 (선택)",
  목적 "매월 갱신되는 KEESS 전체 과정 리스트의 이메일 발송", 항목 "이메일",
  기간 "수신 동의 철회 시까지",
  말미에 "동의하지 않아도 리스트 다운로드는 이용하실 수 있습니다. 발송되는
  이메일에서 언제든 수신을 거부할 수 있습니다." 포함.
- ctpriv1: 목적 "콘텐츠 솔루션 도입 문의의 접수·상담 및 회신",
  항목 "(필수) 담당자명, 회사/기관, 이메일", 기간·거부권리 ldpriv1 준용.

[제출 데이터]
4개 폼 제출 payload에 privacy_agreed(boolean), agreed_at(ISO8601)을 추가한다.
다운로드 모달은 선택 동의가 렌더된 경우에만 marketing_agreed(boolean)를 추가한다.
입력값(이름·연락처·이메일 등)은 분석 도구(GA4 등)로 절대 전송하지 않는다.

[완료 조건]
- 4곳 모두 필수 미동의 시 제출 차단 + 에러 노출, 동의 시 정상 접수
- C 모달: 배너 문구 교체, 선택 미체크 상태에서 다운로드 정상
- C 모달: download.optional=null 설정 시 배너+선택 동의 동시 비노출 확인
- 도입문의 3개 폼은 선택 동의 행 미노출
- /content 모달 2개 간 id·상태 충돌 없음
- 홈 상담·부정훈련 신고 폼 회귀 없음
- 모바일 뷰 레이아웃 정상
```

## 8. G2-11 연계

1. 본 명세의 동의 문구는 가안 — G2-11 정보보호팀 검토(4-2 9번)에 3장 전문 4종+선택 1종을 첨부해 승인받는다.
2. 부정훈련 신고 조회 폼(이름·연락처, 무동의)은 본 빌드 범위 외 — 신고 접수 시 동의로 갈음 가능한지 정보보호팀 판단.
3. 매월 리스트 발송의 운영 주체·프로세스는 영업팀 확인 항목(G2-11 4-1)에 포함 — 미회신·미운영 확정 시 위 7장 히든 스위치로 처리.
4. 개인정보처리방침 페이지(Footer placeholder) 제작은 G2-11 승인 결과를 입력으로 하는 별도 태스크.
